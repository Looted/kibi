## Context

Kibi v0.0.1 stores traceability data and supports validation, but agents need
derived answers (impact, transitive dependencies, coverage gaps) as first-class
tool outputs.

## Decision

Implement Phase 1 inference in Prolog and expose it with explicit MCP tools:
- New predicates in `packages/core/src/kb.pl` for transitive closure, impact,
  coverage gaps, staleness, orphan detection, and ADR conflict/deprecation use.
- MCP tools:
  - `kb_derive` for rule execution
  - `kb_impact` for typed impact results
  - `kb_coverage_report` for aggregate rollups
- Deterministic response shape with explicit structured fields and provenance.

## Consequences

- Agents can ask one-hop tool queries for multi-hop reasoning outcomes.
- Phase 1 behavior is testable with both Prolog and MCP-level tests.
- Future policy/readiness layers can compose existing predicates instead of
  duplicating logic in TypeScript.
