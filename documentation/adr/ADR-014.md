## Context

Kibi previously relied on a mix of manual and ad-hoc scripts for versioning and publishing npm packages (core, cli, mcp). This approach led to inconsistencies, risk of human error, and lack of traceability in release operations.

## Decision

**Adopt [Changesets](https://github.com/changesets/changesets) as the canonical tool for release automation.**

- All versioning, changelog generation, and npm publishing are managed via Changesets workflows
- Release steps are integrated into CI/CD pipelines
- Manual version bumps and direct npm publish are prohibited
- Release policy is documented in Kibi and synced to the KB

## Rationale

- Changesets is a widely adopted, open-source tool for multi-package monorepos
- Provides atomic, auditable release operations with changelog generation
- Integrates with GitHub Actions and other CI/CD systems
- Reduces risk of inconsistent or accidental releases
- Enables traceability and reproducibility for all published artifacts

## Implications

- All contributors must use the documented Changesets workflow for releases
- CI/CD pipelines must enforce Changesets-based release gates
- Documentation and KB must be updated to reflect this policy

## Fallback Guidance

If KB query is unavailable or unreliable, agents and maintainers MUST refer to this ADR and REQ-020 for authoritative release policy. Documentation is the source of truth in case of KB instability.

## Status

Accepted and implemented as of 2026-03-11.
