## Context

Node.js ESM has stricter requirements than Bun:
- No `.ts` imports allowed (must compile to `.js`)
- Imports must include `.js` extension for relative imports
- No `__dirname` in ESM (must use `import.meta.url`)
- JSON imports require `with { type: "json" }` assertion

## Changes Made

### 1. Local imports with .js extension
```typescript
// Before (Bun-only)
import { checkCommand } from "./commands/check";

// After (Node + Bun)
import { checkCommand } from "./commands/check.js";
```

### 2. ESM __dirname equivalent
```typescript
// Added to files that need __dirname
import { fileURLToPath } from "node:url";
const __filename = fileURLToPath(import.meta.url);
const __dirname = path.dirname(__filename);
```

### 3. JSON to TypeScript conversion
- Created `src/public/schemas/entity.ts` - exports schema object directly
- Removed JSON import from MCP package
- No longer needs `with { type: "json" }` assertion
