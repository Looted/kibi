# Requirement: Release Automation with Changesets

All Kibi releases (core, CLI, MCP server) must be automated using the [Changesets](https://github.com/changesets/changesets) workflow. This ensures:

- Consistent semantic versioning across all npm packages
- Automated changelog generation
- Safe, auditable, and reproducible releases
- Integration with CI/CD for publish gates and release verification

## Acceptance Criteria
- All release steps are automated via Changesets
- Manual version bumps and direct npm publish are prohibited
- Release policy is documented in Kibi and synced to the KB
- Fallback: If KB query is unstable, agents must consult documentation/requirements/REQ-020.md and ADR for release policy

## Rationale
Manual release processes are error-prone and inconsistent. Changesets provides a robust, open-source solution for multi-package release management, changelog generation, and CI integration.

## Fallback Guidance
If KB query is unavailable or unreliable, agents and maintainers MUST refer to this document and the corresponding ADR for authoritative release policy.
